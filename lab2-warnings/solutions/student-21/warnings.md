# Отчет о предупреждениях

## Предупреждения -Wall

### 1. Неиспользуемая переменная (`-Wunused-variable`)
- Строка: 136
- Код: `int debug_counter = 0;`
- Текст предупреждения: `warning: unused variable 'debug_counter' [-Wunused-variable]`
- Объяснение: Переменная объявлена и инициализирована значением 0, но нигде не используется в дальнейшем коде.

---

### 2. Неиспользуемая функция (`-Wunused-function`)
- Строка: 20
- Код: `static void debug_log(void) { printf("debug\n"); }`
- Текст предупреждения: `warning: 'debug_log' defined but not used [-Wunused-function]`
- Объяснение: Статическая функция `debug_log` определена, но нигде не вызывается. Компилятор предупреждает о статических функциях, которые не используются, так как они не могут быть вызваны из других единиц трансляции.

---

### 3. Отсутствие оператора return (`-Wreturn-type`)
- Строка: 28–32
- Код: `static int get_version(void) { printf("v1.0"); /* нет return */ }`
- Текст предупреждения: `warning: no return statement in function returning non-void [-Wreturn-type]`
- Объяснение: Функция `get_version` объявлена как возвращающая `int`, но не содержит оператора `return`. Управление доходит до конца функции без возвращения значения.

---

### 4. Неявный проваливающийся переход в switch (`-Wimplicit-fallthrough`)
- Строка: 94–96
- Код:
  ```c
  case 1:
      fall_demo = 2;
      /* нет break — проваливается в case 2 */
  case 2:
  ```
- Текст предупреждения: `warning: this statement may fall through [-Wimplicit-fallthrough=]`
- Объяснение: В конце `case 1` нет оператора `break`, поэтому выполнение продолжается в `case 2`. Это типичная ошибка программиста, поэтому компилятор предупреждает о неявном «провале».

---

### 5. Лишний аргумент в printf (`-Wformat-extra-args`)
- Строка: 140
- Код: `printf("Лабораторная работа №2\n", 0);`
- Текст предупреждения: `warning: too many arguments for format [-Wformat-extra-args]`
- Объяснение: Строка формата `"Лабораторная работа №2\n"` не содержит ни одного спецификатора формата, но функции передаётся лишний аргумент `0`. Этот аргумент игнорируется, но указывает на вероятную ошибку.

---

### 6. Пустое тело оператора if (`-Wempty-body`)
- Строка: 61
- Код: `if (coeff_a > 0) ;`
- Текст предупреждения: `warning: suggest braces around empty body in an 'if' statement [-Wempty-body]`
- Объяснение: Тело оператора `if` состоит только из точки с запятой — пустого оператора. Это может скрывать ошибку: программист мог намереваться написать какой-то код внутри блока.

---

### 7. Отсутствие скобок в составном выражении (`-Wparentheses`)
- Строка: 65
- Код: `int mixed_check = discriminant > 0 || discriminant < 0 && coeff_a != 0;`
- Текст предупреждения: `warning: suggest parentheses around '&&' within '||' [-Wparentheses]`
- Объяснение: Операторы `&&` и `||` смешаны без явных скобок. Из-за приоритета операторов `&&` вычисляется раньше `||`, что может не соответствовать намерению программиста. Правильно: `(discriminant < 0 && coeff_a != 0)` в явных скобках.

---

### 8. Сравнение знакового и беззнакового типов (`-Wsign-compare`)
- Строка: 71
- Код: `for (int iter = 0; iter < max_iter; iter++)`
- Текст предупреждения: `warning: comparison of integer expressions of different signedness: 'int' and 'unsigned int' [-Wsign-compare]`
- Объяснение: Переменная `iter` имеет тип `int` (знаковый), а `max_iter` — тип `unsigned int` (беззнаковый). При сравнении знаковых и беззнаковых типов возможно неожиданное поведение из-за неявного приведения типов.

---

### 9. Использование char как индекса массива (`-Wchar-subscripts`)
- Строка: 80
- Код: `int lookup_val = lookup_table[char_idx];`
- Текст предупреждения: `warning: array subscript has type 'char' [-Wchar-subscripts]`
- Объяснение: Переменная `char_idx` имеет тип `char`, который может быть знаковым и принимать отрицательные значения, что приведёт к обращению к памяти по отрицательному индексу. Для индексации массивов следует использовать тип `int` или `size_t`.

---

### 10. Несоответствие знаковости указателей (`-Wpointer-sign`)
- Строка: 86
- Код: `unsigned char *ustr = equation_str;`
- Текст предупреждения: `warning: pointer targets in initialization of 'unsigned char *' from 'char *' differ in signedness [-Wpointer-sign]`
- Объяснение: Переменная `equation_str` имеет тип `char *`, а `ustr` — `unsigned char *`. Инициализация указателя `unsigned char *` значением `char *` выполняется без явного приведения типов, что может вызвать проблемы при работе со знаковыми символами.

---

## Предупреждения -Wextra (бонус)

### 11. Неиспользуемый параметр функции (`-Wunused-parameter`)
- Строка: 37
- Код: `static void process_data(int value, int extra, int mode)`
- Текст предупреждения: `warning: unused parameter 'extra' [-Wunused-parameter]`
- Объяснение: Параметр `extra` объявлен в сигнатуре функции `process_data`, но нигде внутри функции не используется. Обычно это указывает на незавершённую реализацию или лишний параметр.

---

### 12. Параметр присваивается, но значение не используется (`-Wunused-but-set-parameter`)
- Строка: 37, 39
- Код: `int mode` (параметр), `mode = 1;` (тело функции)
- Текст предупреждения: `warning: parameter 'mode' set but not used [-Wunused-but-set-parameter]`
- Объяснение: Параметру `mode` присваивается новое значение `1`, но это новое значение нигде не читается после присваивания. Начальное значение параметра тоже не используется. Это означает, что вся работа с `mode` внутри функции бессмысленна.

---

### 13. Сравнение, всегда истинное для беззнакового типа (`-Wtype-limits`)
- Строка: 107
- Код: `unsigned int pos_check = 5; if (pos_check >= 0)`
- Текст предупреждения: `warning: comparison of unsigned expression in '>= 0' is always true [-Wtype-limits]`
- Объяснение: Тип `unsigned int` не может принимать отрицательные значения, поэтому условие `pos_check >= 0` всегда истинно. Компилятор предупреждает о бессмысленном сравнении.

---

### 14. Не все поля структуры инициализированы (`-Wmissing-field-initializers`)
- Строка: 113
- Код: `struct CoeffInfo info = {3};`
- Текст предупреждения: `warning: missing initializer for field 'degree' of 'struct CoeffInfo' [-Wmissing-field-initializers]`
- Объяснение: Структура `CoeffInfo` содержит три поля: `count`, `degree`, `is_monic`. При инициализации `{3}` явно задаётся только поле `count = 3`. Поля `degree` и `is_monic` неявно инициализируются нулём. Предупреждение напоминает об возможной ошибке — программист мог забыть о каком-то поле.

---

### 15. Переменная присваивается, но значение не используется (`-Wunused-but-set-variable`)
- Строка: 144–145
- Код: `int temp_storage; temp_storage = 42;`
- Текст предупреждения: `warning: variable 'temp_storage' set but not used [-Wunused-but-set-variable]`
- Объяснение: Переменной `temp_storage` присваивается значение `42`, но это значение никогда не читается в дальнейшем коде. В отличие от `-Wunused-variable`, это предупреждение указывает конкретно на случай, когда запись в переменную есть, а чтения нет.
